# Compliance Schemas

This directory contains the core data models used by the Compliance Evolution Engine.

These schemas define the structured outputs produced after processing SEBI regulatory circulars and serve as the contract between the AI extraction engine, reconciliation engine, backend APIs, and frontend application.

## Files

### obligation_schema.json

Defines the structure of a compliance obligation extracted from a regulatory circular.

Each obligation contains:

- Unique identifier
- Source circular
- Clause reference
- Original regulatory text
- Applicable entities
- Responsible department
- Deadline
- Required evidence
- Priority
- Status
- Confidence score
- Citation metadata

---

### patch_schema.json

Defines the structure of a compliance patch generated by comparing two versions of a regulatory circular.

Each patch captures:

- Type of regulatory change
- Previous and updated circulars
- Affected obligation
- Impact level
- Departments affected
- Clause mapping
- AI reasoning
- Human review status

---

These schemas form the foundation of the Compliance Evolution Engine and ensure consistent, explainable, and auditable compliance intelligence across the platform.
